WorkLedger — Project Setup Guide

Payroll Manager · Next.js 16 · Prisma 5 · Neon PostgreSQL · No server needed for this page

Prerequisites

  • Node.js 18+ (LTS recommended)
  • npm (ships with Node)
  • A Neon project with pooled + direct connection strings (neon.tech)

Step 1 — Clone & install

From the project root:

npm install

Step 2 — Configure .env

Copy .env.example to .env. Use your Neon URLs and a long random secret.

DATABASE_URL="postgresql://USER:PASSWORD@HOST.neon.tech/neondb?sslmode=require"
DIRECT_URL="postgresql://USER:PASSWORD@DIRECT-HOST.neon.tech/neondb?sslmode=require"
AUTH_SECRET="paste-output-of-node_-e_crypto_randomBytes"

Generate a secret:

node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

Step 3 — Run setup

Runs Prisma codegen, applies migrations (migrate deploy uses DIRECT_URL), and seeds demo data.

npm run setup

Step 4 — Start dev server

npm run dev

Open http://localhost:3000. The CLI may run a quick env check first.

Common errors

P1001 / Can’t reach database
Fix: verify DATABASE_URL in .env, Neon project is awake, and the string matches the Neon dashboard (no typos).
EPERM rename / DLL / ENOENT while installing
Fix: close VS Code/Cursor and other terminals, end stray node.exe processes in Task Manager, then reopen and retry npm install.
Migration fails with Neon pooler
Fix: set DIRECT_URL to the direct connection string (hostname without -pooler). Keep DATABASE_URL pooled for the app.
